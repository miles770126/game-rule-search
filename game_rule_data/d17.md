## 배경

수익화 상품 중 일부는 **여러 종류의 하위 상품**으로 구성된다.

<예시>

| 상품명 | 하위 구성 |
| --- | --- |
| 럭키스핀 | 저가형 / 고가형 |
| 침공 1 of 3 | 유저 세그먼트에 따라 A / B / C 상품 (24.4에 적용 예정) |

이러한 상품은 각 하위 상품이 **특정 조건을 충족해야 노출**되는 구조이기 때문에, 일반 플레이 환경에서는 모든 하위 상품을 동시에 확인하기 어렵다.

---

## 문제

Apple iOS 심사(리뷰모드)에서는 **판매 중인 모든 인앱 상품이 화면에 노출**되어야 한다.

그러나 조건 기반으로 등장하는 하위 상품들은 리뷰어 환경에서 자연스럽게 표시되지 않을 수 있어, **심사 거절의 원인**이 될 수 있다.

---

## 대응 방향

리뷰모드 진입 시, 하위 상품 간 **전환 버튼**을 제공하여 심사자가 모든 하위 상품을 직접 확인할 수 있도록 한다.

### 동작 방식

- 리뷰모드에서는 상품 UI 내에 **[저가형 보기] / [고가형 보기]** 형태의 전환 버튼이 노출됨
- 버튼 탭 시 해당 하위 상품으로 UI가 전환되어, 상품 구성·가격·설명 등을 확인 가능
- 일반 플레이 환경(비리뷰모드)에서는 기존 조건 로직 그대로 유지

### 적용 예시 (럭키스핀)

`[ 저가형 보기 ]  [ 고가형 보기 ]
       ↓ 버튼 탭
  해당 가격대의 럭키스핀 상품 UI 표시`

---

## 적용 대상

다음 조건에 해당하는 상품은 본 가이드에 따라 리뷰모드 대응을 구현한다.

- 하위 상품이 2개 이상 존재하며
- 각 하위 상품이 **유저 세그먼트, 레벨, 이벤트 조건 등 특정 조건에 의해 분기**되는 경우

---

## 참고

- 리뷰모드 여부는 기존 iOS 리뷰모드 판별 로직을 그대로 활용
- 전환 버튼 UI 디자인은 럭키스핀 구현 화면을 레퍼런스로 사용
