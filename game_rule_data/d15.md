# [오퍼] five pack 유저 세그먼트별 보상 작업 및 치트 참고

작업자: [정명진] 지누

---

설명

- five pack 오퍼부터 **유저 세그먼트(저/중/고)별**로 **다른 보상을 제공**합니다.
    - *세그먼트 기준은 **누적 구매액**
- 본 문서는 각 세그먼트별 보상을 데이터 시트에 반영하고, 개발 치트 기능으로 검증할 때 **참고할 사항**을 정리합니다.

1. 유저 세그먼트별 데이터 작업 방식

[세그먼트 분류 프로세스]

- 구글 문서: [SP24 세그먼트 참고(14p)](https://docs.google.com/presentation/d/1Je_tBwytawmS_RBfIwBy1C1QqX9I4ehLYk9V_7T0CCg/edit?usp=sharing)

[개발자]

- 사업팀에서 제공한 범위 확인
- 누적 구매액 기준으로 세그먼트 그룹 분류

[기획자]

- 저/중/고 세그먼트별 보상 구성 검토
- 각 세그먼트에서 정상 데이터가 출력되는지 테스트 검증

[보상 구조 원칙]

- 구글 문서: [SP24 보상 참고(15p)](https://docs.google.com/presentation/d/1Je_tBwytawmS_RBfIwBy1C1QqX9I4ehLYk9V_7T0CCg/edit?usp=sharing)
- 세그먼트당 보상 항목 : 최대 5개 이하로 제한 
*UIUX 상 고려한 내용으로 보상 종류가 많아질 경우 처리 방향성 확인 필요
- 각 세그먼트의 보상 구성이 과도하지 않도록 관리

2. 보상량 변경 데이터 구조

- 구글 문서: [데이터 구조 설명 참고(38p)](https://docs.google.com/presentation/d/1Je_tBwytawmS_RBfIwBy1C1QqX9I4ehLYk9V_7T0CCg/edit?usp=sharing)
- 유저 세그먼트별로 보상 구성을 변경할 수 있는 시스템 구조 작업 시 참고
    - 각 세그먼트별로 다른 보상을 제공 가능
    - 데이터만 교체하여 유연하게 변경 가능

데이터 처리 방식

Step 1 : FivePackGiftRewardData 시트에서 `reward_day_group` 조정

- 실제 보상 내용 수정

Step 2: FivePackGiftSegmentData 시트에서 최종 수정

- 세그먼트별 그룹 ID 매핑 확인

3. 치트 테스트 방식

개발 기간 테스트 가능 기능

세그먼트별 화면 확인 :

- 저/중/고 오퍼를 선택하여 각 세그먼트별 화면 직접 확인 가능

내부 동작 방식 검증 :

- 데이터가 정상 출력되는지 검증 가능

QA 기간 추가 가능 기능 (예정)

날짜 넘어가는 기능 

- 쿨타임 단축 기능으로 쿨타임 종료 시 즉시 다음 오퍼가 노출되도록 검증하고자 함
- QA 기간에 다양한 시나리오 검증 가능
